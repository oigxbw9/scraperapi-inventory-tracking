## 实际搭建库存监控的几个经验

**先用免费额度验证你的目标站能不能通。** 不同站点的反爬难度差异很大。我之前想监控某个小众潮牌官网，发现它用了比较冷门的人机验证方案，普通请求过不去，得开render参数才行。5000次免费调用足够你把目标站全部测一遍。

**JS渲染请求消耗额度是普通请求的10倍。** 这个很多人刚上手会忽略。如果你的目标站不需要JS渲染就能拿到库存数据，千万别开render参数，白浪费额度。我建议先用浏览器开发者工具看一下，库存字段是不是在初始HTML里就有。

**设计好你的监控频率。** 不是越频繁越好。大部分商品的库存变动不会精确到分钟级别，每15-30分钟查一次对绝大多数场景够用了。频率太高除了费额度，还容易触发目标站的频率限制。

**做好异常处理和告警分级。** 我的脚本里区分了三种状态变化：补货（从无货到有货）、断货（从有货到无货）、价格变动。补货告警优先级最高，直接推送；价格小幅波动我只记录日志，不推送，避免告警疲劳。
